# Error Codes

PreShip uses standard HTTP status codes and returns structured error responses. Every error response includes a `success: false` flag, an error code, and a human-readable message.

---

## Error Response Format

All error responses follow this structure:

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "A human-readable description of the error.",
    "details": {}
  }
}
```

The `details` field is optional and provides additional context for certain error types (e.g., validation errors).

---

## HTTP Status Codes

### 400 Bad Request

The request body or query parameters are invalid.

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request body.",
    "details": {
      "fields": [
        {
          "field": "url",
          "message": "Must be a valid URL starting with http:// or https://"
        },
        {
          "field": "checks",
          "message": "Must be an array containing one or more of: accessibility, performance, security, seo"
        }
      ]
    }
  }
}
```

**Common causes:**
- Missing required fields (`url`)
- Invalid URL format
- Invalid check type in `checks` array
- `pages` value exceeds plan limit
- Malformed JSON body

---

### 401 Unauthorized

The request is missing authentication credentials or the provided credentials are invalid.

```json
{
  "success": false,
  "error": {
    "code": "AUTHENTICATION_REQUIRED",
    "message": "A valid API key or Bearer token is required to access this endpoint."
  }
}
```

**Common causes:**
- Missing `X-API-Key` or `Authorization` header
- Invalid or revoked API key
- Expired JWT token
- Malformed authorization header

---

### 403 Forbidden

The authenticated user does not have permission to perform this action.

```json
{
  "success": false,
  "error": {
    "code": "FORBIDDEN",
    "message": "Your current plan does not include this feature. Upgrade to Pro to access scheduled scans."
  }
}
```

**Common causes:**
- Feature not available on current plan (e.g., scheduled scans on Free plan)
- Attempting to access another user's resources
- API key does not have the required permissions

---

### 404 Not Found

The requested resource does not exist.

```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Scan scan_invalid123 not found."
  }
}
```

**Common causes:**
- Invalid scan or project ID
- Resource has been deleted
- Typo in the endpoint URL

---

### 429 Too Many Requests

Rate limit or usage limit exceeded.

#### Rate Limit Exceeded

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Rate limit exceeded. Please retry after 45 seconds.",
    "retryAfter": 45
  }
}
```

#### Usage Limit Exceeded

```json
{
  "success": false,
  "error": {
    "code": "USAGE_LIMIT_EXCEEDED",
    "message": "Monthly scan limit reached (50/50). Upgrade to Pro for 500 scans/month.",
    "limit": 50,
    "used": 50,
    "resetsAt": "2025-02-01T00:00:00.000Z"
  }
}
```

See [Rate Limits](/api-reference/rate-limits) for details on limits by plan and retry strategies.

---

### 500 Internal Server Error

An unexpected error occurred on the server.

```json
{
  "success": false,
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "An unexpected error occurred. Please try again later.",
    "requestId": "req_abc123def456"
  }
}
```

If you encounter persistent 500 errors, contact support with the `requestId` for investigation.

## Error Handling Examples

### JavaScript

```javascript
async function createScan(url) {
  const response = await fetch("https://api.preship.dev/api/v1/scans", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-API-Key": process.env.PRESHIP_API_KEY,
    },
    body: JSON.stringify({ url }),
  });

  const body = await response.json();

  if (!body.success) {
    switch (response.status) {
      case 400:
        console.error("Validation error:", body.error.details?.fields);
        break;
      case 401:
        console.error("Invalid API key. Check your PRESHIP_API_KEY.");
        break;
      case 429:
        if (body.error.code === "RATE_LIMIT_EXCEEDED") {
          const wait = body.error.retryAfter || 60;
          console.log(`Rate limited. Retrying in ${wait}s...`);
          await new Promise((r) => setTimeout(r, wait * 1000));
          return createScan(url); // Retry
        } else {
          console.error("Usage limit reached:", body.error.message);
        }
        break;
      default:
        console.error("Unexpected error:", body.error.message);
    }
    return null;
  }

  return body.data;
}
```

### Python

```python
import requests
import time

def create_scan(url):
    response = requests.post(
        "https://api.preship.dev/api/v1/scans",
        headers={
            "Content-Type": "application/json",
            "X-API-Key": "sk_live_your_api_key_here",
        },
        json={"url": url},
    )

    body = response.json()

    if not body["success"]:
        error = body["error"]

        if response.status_code == 400:
            print(f"Validation error: {error.get('details', {}).get('fields', [])}")
        elif response.status_code == 401:
            print("Invalid API key. Check your PRESHIP_API_KEY.")
        elif response.status_code == 429:
            if error["code"] == "RATE_LIMIT_EXCEEDED":
                wait = error.get("retryAfter", 60)
                print(f"Rate limited. Retrying in {wait}s...")
                time.sleep(wait)
                return create_scan(url)  # Retry
            else:
                print(f"Usage limit reached: {error['message']}")
        else:
            print(f"Unexpected error: {error['message']}")
        return None

    return body["data"]
```

## Error Code Reference

| HTTP Status | Error Code | Description |
|-------------|------------|-------------|
| 400 | `VALIDATION_ERROR` | Request body or parameters failed validation |
| 400 | `INVALID_URL` | The provided URL is not valid or not reachable |
| 401 | `AUTHENTICATION_REQUIRED` | Missing or invalid credentials |
| 401 | `TOKEN_EXPIRED` | JWT token has expired |
| 403 | `FORBIDDEN` | Insufficient permissions or plan limitations |
| 404 | `NOT_FOUND` | Resource does not exist |
| 409 | `CONFLICT` | Resource already exists (e.g., duplicate project URL) |
| 422 | `UNPROCESSABLE_ENTITY` | Request is syntactically valid but semantically invalid |
| 429 | `RATE_LIMIT_EXCEEDED` | Too many requests in the current time window |
| 429 | `USAGE_LIMIT_EXCEEDED` | Monthly scan or page limit reached |
| 500 | `INTERNAL_ERROR` | Unexpected server error |
| 502 | `SCAN_FAILED` | The scan engine failed to process the URL |
| 503 | `SERVICE_UNAVAILABLE` | The service is temporarily unavailable |
